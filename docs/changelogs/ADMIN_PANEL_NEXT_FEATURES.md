# SmartNShine Admin Panel Next Features

This roadmap captures the next feature ideas for the SmartNShine admin panel after the dedicated user inspection page.

## Recommended Next Build

Subscription Management inside the user detail page.

This naturally completes the "inspect whole user" workflow and gives admins/support a practical way to understand and resolve account, quota, and billing issues from one place.

## High Priority

1. User detail page polish: filters inside AI usage, resumes, and interviews.
2. Admin global search: search users, resumes, payments, and AI usage from one bar.
3. Subscription management: view, extend, cancel, manually activate, and downgrade users.
4. Payment/revenue inspection: per-user payment history, failed payments, and receipts.
5. Admin activity audit trail: better filters, target user links, and export logs.
6. User support notes: admins can add internal notes to a user profile.
7. Account risk flags: mark suspicious, high-cost, quota-abusing, or support-needed users.
8. AI usage cost dashboard: cost by user, feature, provider, day, and model.
9. Quota override system: temporary quota boosts without changing subscription tier.
10. Admin notifications: users near quota, failed AI requests, and payment failures.

## Medium Priority

11. Resume inspection page: view resume metadata, template, score, and creation source.
12. Interview analytics: completion rate, average score, and abandoned sessions.
13. Contact/support inbox upgrade: assign status, priority, and admin owner.
14. Feedback management upgrade: tag feedback, mark roadmap candidates, and link feedback to users.
15. Template analytics: which templates are used or downloaded most.
16. Feature usage dashboard: ATS scans, AI generation, extraction, and interview usage.
17. User lifecycle dashboard: new, active, inactive, paid, and churn-risk users.
18. Admin export tools: CSV export for users, payments, AI usage, and contacts.
19. System health panel: API status, AI provider status, and database status.
20. Error monitoring view: recent backend/frontend errors grouped by feature.

## Advanced / Later

21. Admin impersonation mode: view the app as a user with strict audit logging.
22. User segmentation: free, paid, student, inactive, high AI usage, and other saved segments.
23. Campaign/coupon management: discount codes and promotional plans.
24. Bulk user actions: disable, email, export, or reset counters for selected users.
25. Role-based admin permissions: support admin, finance admin, and super admin.
26. AI prompt/version analytics: compare success, cost, and latency by prompt version.
27. Abuse detection: unusual signups, repeated failed AI calls, and spam contact forms.
28. Retention dashboard: cohort charts and subscription conversion funnel.
29. In-app announcements: show notices to selected users or plans.
30. Admin task center: unresolved messages, failed payments, quota issues, and flagged users.
