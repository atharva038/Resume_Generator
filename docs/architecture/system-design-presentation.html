Architecture Overview
High-level system layers, ports, and communication patterns
🖥️
Frontend
React 18 + Vite 6 running on port 5173. TipTap rich text editor, Framer Motion animations, TailwindCSS styling.
⚙️
Backend
Node.js + Express 4 on port 5000. ES Modules throughout. MVC + Service Layer pattern.
🤖
AI Services
Google Gemini (resumes), OpenAI GPT-4o (interviews), Whisper STT (voice input), Chatterbox TTS (voice output).
🗄️
Database
MongoDB Atlas via Mongoose 8. 8 core models with embedded sub-documents for performance.
💳
Payments
Razorpay integration with HMAC-SHA256 signature verification and webhook support.
🔐
Auth
JWT (7-day expiry) + Passport.js for Google & GitHub OAuth. Role-based access (user/admin).

Known Issues & Improvements
Current bugs, technical debt, and recommended improvements

Known Bugs

1
Duplicate GET /tts/health route
voice.routes.js defines the same route at line 55 AND line 141 — the second definition shadows the first silently.
2
Dead ElevenLabs route references
Routes /tts/synthesize-json, /tts/stream, /tts/test reference undefined elevenlabsService — potential runtime crash if reached.
3
Monolithic AIInterview.jsx (3300+ lines)
The entire interview UI is in one file. Hard to maintain, test, and debug. Should be split into sub-components: QuestionCard, VoiceRecorder, EvaluationPanel, etc.
4
No WebSocket for voice interview
Voice interview uses polling/HTTP rather than WebSocket, adding latency to each voice exchange. Should use Socket.io or WebRTC for real-time flow.
5
Interview timer lacks grace period
Auto-complete fires abruptly when timer hits zero — no 10-second warning or grace period for the user to finish their sentence.

Recommended Improvements

AreaImprovementPriority
VoiceReplace HTTP polling with WebSocket (Socket.io) for real-time voice interviewHigh
FrontendSplit AIInterview.jsx into focused sub-componentsHigh
RoutesRemove duplicate /tts/health and dead ElevenLabs routesHigh
CachingAdd Redis for session data and AI response cachingMedium
TestingAdd Jest unit tests for services + Playwright E2E testsMedium
CI/CDAdd GitHub Actions pipeline for automated testing + deploymentMedium
MonitoringAdd Sentry for error tracking, Datadog for API metricsLow
SecurityAdd CSRF tokens for form submissionsLow
